## 1. 产品概述

课程物品准备页是一款纯前端培训活动物资管理工具，用于在培训活动开始前整理签到用品、练习材料、计时器、桌签和备用设备等物资。面向管理员、普通用户和审计员三种角色，实现物资的准备进度追踪、缺口自动检查与变更审计，确保课前物资零遗漏。

- 核心价值：以"缺口驱动"理念管理课前准备流程，自动识别数量不足、责任人缺失、物品重复和课程缺口过多等问题，降低人为遗漏风险
- 目标用户：培训活动组织者、课程管理员、现场协调员

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 管理员 | 角色选择器切换 | 创建/编辑/删除物品、管理分类、批量操作、查看缺口与变更记录、导出 |
| 普通用户 | 角色选择器切换 | 提交准备进度、查看物品列表、查看筛选结果、标记状态 |
| 审计员 | 角色选择器切换 | 查看缺口分析、查看变更记录、导出，不可修改任何数据 |

### 2.2 功能模块

1. **物品管理页**：物品列表、筛选栏、详情面板、批量操作工具栏、自动检查提示
2. **分类管理弹窗**：分类的增删改（仅管理员）
3. **缺口分析面板**：数量不足、责任人缺失、同名重复、课程缺口过多的汇总
4. **变更记录面板**：所有数据变更的时间线记录（仅审计员和管理员可查看）
5. **课前核对模式**：仅展示缺口项和待处理项的精简视图
6. **导出功能**：将当前清单导出为 CSV 文件

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 物品管理页 | 角色选择器 | 页面顶部切换当前角色，角色变更后自动调整可操作范围 |
| 物品管理页 | 筛选栏 | 按课程、分类、责任人、状态、缺口等级筛选，支持组合筛选 |
| 物品管理页 | 物品列表 | 展示所有物品卡片/行，支持多选，显示状态标签和缺口标记 |
| 物品管理页 | 详情/编辑面板 | 右侧或弹窗形式，展示物品完整信息，支持编辑（管理员/普通用户） |
| 物品管理页 | 批量操作栏 | 选中多条后出现，支持批量标记"待准备""已备齐""需补充""暂停使用" |
| 物品管理页 | 自动检查提示 | 实时显示数量低于最低数量、责任人缺失、同名物品重复、同课程缺口过多的警告 |
| 物品管理页 | 课前核对模式开关 | 切换为精简视图，只展示缺口项和待处理项 |

| 缺口分析面板 | 缺口汇总 | 按类别汇总所有问题项，显示缺口等级（严重/警告/提示） |
| 变更记录面板 | 变更时间线 | 按时间倒序展示所有增删改操作记录 |
| 物品管理页 | 导出按钮 | 导出当前筛选后的清单为 CSV |

## 3. 核心流程

### 3.1 物品准备主流程

管理员创建分类和物品 → 普通用户更新准备进度 → 系统自动检查缺口 → 审计员查看缺口与变更 → 课前核对模式确认 → 导出清单

```mermaid
flowchart TD
    "管理员创建分类" --> "管理员添加物品记录"
    "管理员添加物品记录" --> "普通用户填写准备进度"
    "普通用户填写准备进度" --> "系统自动检查缺口"
    "系统自动检查缺口" --> "有缺口?"
    "有缺口?" -->|"是"| "提示补充/调整"
    "有缺口?" -->|"否"| "课前核对模式"
    "提示补充/调整" --> "普通用户更新状态"
    "普通用户更新状态" --> "系统自动检查缺口"
    "课前核对模式" --> "导出清单"
    "审计员查看缺口与变更" --> "课前核对模式"
```

### 3.2 批量操作流程

多选物品 → 点击批量状态按钮 → 确认变更 → 更新状态 → 选中项若被筛掉则清空详情区并提示

### 3.3 筛选联动流程

设置筛选条件 → 列表实时过滤 → 若当前选中项不在筛选结果中 → 详情区自动清空并显示提示信息

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深青色 (#0F766E) 作为品牌色，搭配暖橙 (#EA580C) 作为警示/缺口强调色
- **辅助色**：石板灰 (zinc) 系列用于背景与文字层级
- **按钮风格**：圆角 8px，主按钮实色填充，次按钮描边，危险按钮暖橙色
- **字体**：中文使用系统字体栈，标题使用较粗字重；数字/代码使用等宽字体
- **布局**：桌面端采用左侧列表 + 右侧详情面板的经典双栏布局，顶部工具栏
- **图标风格**：线性图标（lucide-vue-next），与文字对齐

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 物品管理页 | 顶部工具栏 | 角色切换芯片组、课前核对模式开关、导出按钮、缺口分析按钮、变更记录按钮 |
| 物品管理页 | 筛选栏 | 课程下拉、分类下拉、责任人下拉、状态下拉、缺口等级下拉，横向排列 |
| 物品管理页 | 物品列表 | 左侧占 60%，卡片/紧凑行列表，每行显示名称、分类标签、状态徽章、缺口图标 |
| 物品管理页 | 详情面板 | 右侧占 40%，表单式布局，各字段竖排，底部操作按钮 |
| 物品管理页 | 批量操作栏 | 列表底部浮动栏，显示选中数量，四个状态按钮 |
| 物品管理页 | 自动检查提示 | 列表上方横幅，按严重程度着色，可折叠 |
| 分类管理弹窗 | 弹窗 | 居中弹窗，分类列表 + 新增按钮，每项可编辑/删除 |
| 缺口分析面板 | 侧滑面板 | 右侧滑出，按类别分组，每项显示问题描述和跳转链接 |
| 变更记录面板 | 侧滑面板 | 右侧滑出，时间线布局，每条显示操作类型、操作人、时间、变更详情 |

### 4.3 响应式设计

- 桌面端（≥1024px）：双栏布局，筛选栏水平展开
- 平板端（768-1023px）：单栏布局，详情面板改为底部弹窗
- 移动端（<768px）：全屏列表，筛选折叠为按钮触发的抽屉

### 4.4 动效设计

- 列表项选中/取消：平滑过渡
- 详情面板展开/收起：滑入滑出
- 批量操作栏：底部滑入
- 缺口提示：脉冲动画吸引注意
- 课前核对模式切换：列表淡入淡出过渡
